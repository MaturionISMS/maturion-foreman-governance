# Global Autonomy Orchestrator (GAO) Overview

## Purpose
The Global Autonomy Orchestrator (GAO) is the master coordinator that unifies all autonomy subsystems into a coherent, situationally aware, self-correcting autonomous engineering system.

## Core Responsibilities

### 1. Global Decision Making
- Coordinate all autonomy layers
- Prioritize tasks across the entire platform
- Make strategic execution decisions
- Balance resources and risks

### 2. Multi-System Coordination
- Integrate Autonomy Runtime (AUTO-01)
- Coordinate Wave Execution Engine (AUTO-02)
- Utilize Build Recovery Engine (AUTO-03)
- Leverage Multi-Repository Autonomy (AUTO-04)
- Orchestrate Memory System (UML-01)
- Monitor via Observability (OBS-01)
- Notify through Notification System (NOTIF-01)

### 3. Governance Enforcement
- Apply CS1-CS6 across all operations
- Enforce TED (Technology Evolution Doctrine)
- Maintain SBHC (Safety, Boundaries, Health, Compliance)
- Respect embodiment boundaries
- Ensure tenant isolation
